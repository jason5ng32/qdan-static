Windows 10 兼容的软件/应用/扩充/插件生态圈

Windows 10 正式开启微软『有容乃大』战略的核心精随：兼容各大软件应用生态圈。野心巨大，视野深远，但也是风险极高的一步棋。BUILD 大会是针对开发者程序员的活动，提供技术开发上的愿景，但对一般用户而言，最实际的是 Windows 10 上能干啥事？以下细数用户可以期待在 Window 10 使用到那些软件/应用/扩充/插件。

  1. Windows 桌面软件（Windows Software）

    Windows XP ～ Windows 7 也能跑上的桌面软件（俗称：exe档），这也是微软最深厚的底气，难以取代的关键。Windows 10 照样跑上 exe 为中心的所有软件，所以以往习惯常用的软件大可放心继续用。Windows 桌面软件有多少个？由于无应用商店（ = 无审核管理机制），多方数据统计落差大，只能说规模庞大难以估计。

  2. Windows 桌面/平板应用（Windows Apps）

    Windows 应用商店裡的应用（可以想做是动态砖/Metro设计风格的桌面软件），微软在 Windows 8 与 8.1 上将桌面/平板应用商店（Windows Apps）与手机应用商店（Windows Phone Apps）分为两个市集（应用出版平台）， Windows 10 目前看来是要整合成为一个 Windows 10 应用商店。 总之 Windows 应用商店裡头声称有近60万个桌面+平板+手机应用（根据 Microsoft by the Numbers ，目前应用商店裡有58万5千个应用）

  3. Windows 手机应用（Windows Phone Apps）

    Windows 手机上的应用，透过 Continuum 功能 ，Windows 10 手机只要连上屏幕既可变身为一台 mini PC ，将手机内的应用转变成桌面版本的 UI 用户界面。再透过无线蓝芽鼠标与键盘，建立出完全桌面软件的操作环境。

  4. iPhone/iPad 应用 (iOS Apps)

    微软提供了开发者工具，让 iOS 上的应用转译成 Windows 应用，使得 iOS 的应用开发商更快速将自己在 iOS 的应用转译发佈到 Windows 应用商店上，补足 Windows 上应用不足的现况。微软提供的开发者转译工具的成熟度以及开发商意愿将会是关键，不确定因素颇高。持续关注是否会有一波 iOS 应用透过此捷径登上 Windows 10。

    2014年统计 iOS 上有 120 万个应用。

  5. 安卓应用（Android Apps）

    同上。微软提供开发者直接将安卓 apk 应用转译发佈到 Windows 应用商店的工具。安卓生态圈规模巨大，也值得期待是否会有一波安卓应用登上 Windows 10 应用商店。

    2014年统计 Android 共有 130 万个应用（不清楚此数据是否包含 Google Play 商店以外的应用）

  6. Edge 扩充/插件（Edge Extensions）

    概念是让开发者将其在 Chrome 上的扩充与 Firefox 上的附加元件打包到 Windows Edge 平台上来发布，估计会以应用商店的面貌呈现，目前还不清楚会是 Edge 独立出来一个应用商店还是整合到 Windows 10 应用商店裡头。与前排的 iOS/安卓开发者不同，Edge 扩充/插件无须再次转译（而是相同的 Web 编程语言），开者发只是多了一个平台发布，估计意愿会相当高。应该不久后能看到大量的 Chrome Extensions 与 Firefox Add-ons 会有 Edge 版本。（Edge 作为 Windows 10 的预载浏览器，市佔率攀升应该会颇快，原 IE 用户基本上能够转移过来，个人推估 Edge 应该会威胁到不少国内浏览器的应用/扩充/插件中心的发展，毕竟如此不易用的 IE 在国内都还是浏览器领头羊，用户基础盘大）

  7. Office 应用商店（Office Apps / Outlook Add-ins）

    桌面版的 Office 其实就已经存在（话说使用率不高），这 Office 应用商店的 Apps 与 Add-ins 与以往不同，拥抱 Web 语言并且深度整合微软旗下服务的帐号，使其能够与 OneDrive, Outlook, Office 365 服务在 Web 上更一统化，相信会有更多开发商与微软整合。说实在话，一看放去都是实用度相当高的效率工具，但其最大市场先进者是 Google Drive Add-ons ，众多开发商都已经高度依赖谷歌帐号服务：Gmail、Google Drive（Docs/Sheets/Slides）、Google+ 等开发提供解决方案。另外 Office Apps 独立于 Windows 应用商店自成一个 Office 应用商店，估计是为了并存在 iOS 与安卓的生态圈。

    （注：非 Windows 10 操作系统独有，而是透过桌面版浏览器）

    • 在线网页应用（Web Apps）

      以前的网页多为静态，是纯粹的信息获取，现在的网页大多本身是个平台，不单是交互性，更提供传统桌面软件的『功能』，以桌面版浏览器作为媒介，即可登上的网页应用 Web Apps ，是新一代软件应用的趋势，越来越多应用开发者首选在 Web 端作为试水温的场所，网页应用的选择最多元无限，高度依赖互联网，所有的应用皆需要在线联网。

      （注：非 Windows 10 操作系统独有，而是透过桌面版浏览器，移动端浏览器上目前还是较为受限）

    • Chrome 离线应用（Chrome Offline Apps）

      谷歌补足网页应用必需联网在线的解决方案，在 Chrome 应用商店提供了离线版的网页应用，透过 Chromium 的各牌浏览器（Chrome 是谷歌牌的 Chromium 浏览器），离线也能使用 Chrome 商店裡的离线应用，目前为数不多，但都是高质量实用的工具为主。

      （注：非 Windows 10 操作系统独有）

    • Google Drive 连结应用（Google Drive Add-ons）

      在 Chrome 应用商店勾选『Works with Google Drive』则会看到一系列与谷歌帐号服务整合的应用，在 Docs/Sheets/Slides 则可从 Add-ons 的工具列添加安装，谷歌是个帐号大宝箱，再加上其 Drive, Gmail, Calendar 等效率工具的整合（Facebook 脸谱也是帐号大仓库，但缺乏谷歌的效率工具，多了谷歌缺乏的社交性），Office 应用市集也开始从 Web 下手，与谷歌一样整合其帐号服务+效率工具，乐见其成。

      （注：非 Windows 10 操作系统独有，而是透过桌面版浏览器）

    • Chrome 扩充（Chrome Extensions）

      谷歌应用商店的 crx 扩充，同样只需透过 Chromium 的各牌浏览器即可使用。Chrome 扩充的生态圈裡有众多实用的小工具，真心推荐花时间去探索。

      （注：非 Windows 10 操作系统独有）

    • Firefox 附加元件（Firefox Add-ons）

      Chrome 扩充概念的原型，透过 html+js 编程小工具的始祖，透过火狐浏览器即可使用，火狐附加元件的生态圈相当庞大（虽然近来也都有相同的 Chrome 扩充版本，与 Chrome 扩充重叠性高），严守开源精神的火狐还是有需多爱戴者为其开发附加元件，是个值得花时间去探索的世界。

      （注：非 Windows 10 操作系统独有，而是透过火狐桌面版浏览器）

    • HoloLens 全息眼镜 + 可穿戴式 + 物联网 + 机器人（HoloLens & Wearable & IoT & Robot Apps）

      微软此次 BUILD 大会确实有些许展示 HoloLens 上的应用，也有轻轻提及与 Raspberry Pi + Arduino 的相容，虽目前信息有限，但估计将自身系统大幅度轻量化的 Windows 10 在虚拟现实、全息体感、可穿戴式、物联网、机器人的路上都已经铺好了砖，可预期接下来的相关应用。

    • Xbox One / Live 应用

      微软在电视、游戏的娱乐接口，有许多游戏与娱乐应用。笔者非 Xbox 玩家，在此不多言。
